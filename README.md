# 탄소 배출 관리 시스템 (Carbon Emission Management System)

## 📌 프로젝트 개요

기업 활동 데이터(전기, 원소재, 운송 등)를 입력하면
배출계수 기반으로 탄소 배출량(kgCO2e)을 자동 계산하고 시각적으로 관리할 수 있는 웹 애플리케이션입니다.

사용자는 데이터를 입력하고, 카테고리별 배출량과 총 배출량을 직관적으로 확인할 수 있습니다.

---

## 🛠 기술 스택

* Frontend: React, TypeScript, Vite
* State Management: useState, useMemo
* Chart: Recharts
* Version Control: Git, GitHub

---

## 🚀 주요 기능

### 1. 활동 데이터 입력

* 날짜, 활동 유형(전기 / 플라스틱1 / 플라스틱2 / 운송), 설명, 수량 입력
* 입력값 validation 처리 (빈 값 및 0 이하 값 방지)

### 2. 탄소 배출량 자동 계산

* 활동 유형별 배출계수 적용하여 배출량 계산

### 3. 카테고리별 요약

* 전기 / 원소재 / 운송 각각의 배출량 표시
* 총 배출량 계산 및 강조 표시

### 4. 데이터 테이블 관리

* 입력된 데이터 리스트 표시
* 개별 데이터 삭제 기능

### 5. 데이터 시각화

* 카테고리별 배출량 막대 그래프 제공

### 6. 샘플 데이터 기능

* 버튼 클릭으로 테스트 데이터 자동 입력

---

## 🧱 배출계수 관리 구조

현재 배출계수는 DB 테이블 구조를 고려하여 다음과 같이 설계했습니다.

| activityType | factor | inputUnit | version | source |
| ------------ | ------ | --------- | ------- | ------ |
| electricity  | 0.456  | kWh       | v1      | 과제 제공  |
| plastic1     | 2.3    | kg        | v1      | 과제 제공  |
| plastic2     | 3.2    | kg        | v1      | 과제 제공  |
| transport    | 3.5    | ton-km    | v1      | 과제 제공  |

👉 현재는 프론트엔드에서 관리하지만
👉 향후 DB 테이블로 확장 가능한 구조로 설계되었습니다.

---

## 🔄 배출계수 Version 관리

배출계수는 시간에 따라 변경될 수 있기 때문에
다음과 같은 필드를 통해 이력 관리가 가능하도록 설계했습니다.

* version
* validFrom
* validTo
* source

현재는 구조만 반영하였으며,
향후 DB에서 이력 기반 조회가 가능하도록 확장 가능합니다.

## 🧠 설계 의도 및 판단

### 1. 상태 관리 구조

* 모든 데이터는 activities 배열로 통합 관리
* useMemo를 활용하여 계산 결과 자동 갱신

👉 데이터 변경 시 성능 저하 없이 재계산

---

### 2. 배출계수 구조 분리

* emissionFactorTable 구조로 관리

👉 이유:

* 하드코딩 제거
* DB 테이블 구조로 확장 가능
* 유지보수 용이

---

### 3. UI 구조 설계

* 입력 / 요약 / 시각화 영역 분리

👉 이유:

* 데이터 흐름 직관화
* SaaS 형태 UI 구조 반영

---

## ⚖️ 설계 Trade-off

* 단순 구현 vs 확장성

초기 과제에서는 빠른 구현을 위해 프론트엔드 중심으로 개발했지만,
배출계수 구조는 DB 테이블 형태로 확장 가능하도록 설계하여
실제 SaaS 환경에서도 적용 가능하도록 고려했습니다.
